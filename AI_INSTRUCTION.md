# Drunken-Team AI Core Instructions: Permission & Environment Tiers

เอกสารนี้คือกฎระเบียบขั้นเด็ดขาด (Core Directives) สำหรับ AI Agent ทุกตัวในทีม Drunken-Team ไม่ว่าจะรันอยู่บนเครื่องของผู้ใช้คนใด หรือระบบปฏิบัติการใด (macOS, Windows, Linux)

AI ต้องทำงานตาม **Permission Tiers** ด้านล่างนี้อย่างเคร่งครัด เพื่อให้ระบบทำงานได้แบบอัตโนมัติ ราบรื่น และไม่ถูกขัดจังหวะจากการขอรหัสผ่านหรือการสแกนนิ้วของ OS (Zero Friction Automation)

---

## 🎯 The Permission Tiers

### 🟢 Tier 0: Zero Friction (พื้นที่ปลอดภัย - ทำงานได้ทันที)
นี่คือเป้าหมายหลัก 90% ของงานทั้งหมดต้องอยู่ใน Tier นี้ AI สามารถจัดการเองได้ 100% โดยไม่เกิดการแจ้งเตือนจาก OS
- **สิทธิ์การทำงาน:** ทำงานภายใต้สิทธิ์ของ User ปัจจุบันเท่านั้น (User Space)
- **การเข้าถึงไฟล์:** อ่าน/เขียน/สร้างไฟล์ ภายในโฟลเดอร์โปรเจกต์ หรือโฟลเดอร์ที่ได้รับอนุญาตแล้วเท่านั้น
- **การจัดการ Environment:**
  - **Python:** บังคับใช้ Virtual Environment (`.venv`) หรือ `uv` ภายในโปรเจกต์ **ห้ามใช้ระบบ Python ของ OS เด็ดขาด**
  - **Node.js:** บังคับติดตั้งผ่าน `npm install` ในโปรเจกต์ (Local `node_modules`) **ห้ามใช้แฟล็ก `-g` เด็ดขาด**
- **Network:** อนุญาตให้เชื่อมต่อ API ภายนอก, Pull/Push Git ได้ปกติ

### 🟡 Tier 1: OS / App Consent (ขออนุญาตครั้งแรกครั้งเดียว)
เมื่อ AI พยายามเข้าถึงทรัพยากรระดับ OS ครั้งแรก (เช่น กล้อง, ไมค์, โฟลเดอร์ Documents/Desktop บน Mac)
- **กฎของ AI:** AI ต้องแจ้งเตือนผู้ใช้ล่วงหน้าว่า "อาจจะมีหน้าต่าง Pop-up เด้งขึ้นมาให้กด Allow จากระบบปฏิบัติการ"
- AI ไม่สามารถกด Allow แทนผู้ใช้ได้ ต้องรอให้ผู้ใช้กดยืนยัน

### 🟠 Tier 2: Team Protocol (กฎเหล็กของทีม - ขออนุญาตจาก Boss)
กฎเฉพาะของ Drunken-Team เมื่อต้องจัดการกับ **Destructive Actions**
- **ครอบคลุมถึง:** คำสั่งลบไฟล์ (`rm`, `rm -rf`), ลบ Database, หรือการเปลี่ยนแปลงโครงสร้างระดับใหญ่
- **กฎของ AI:**
  1. **ห้าม** รันคำสั่งเหล่านั้นทันที
  2. ต้องสรุปการกระทำเป็นตาราง Markdown ให้ชัดเจน
  3. บังคับเรียกใช้สคริปต์ `python scripts/ask_boss.py "<ข้อความคำถาม>"`
  4. หยุดรอจนกว่า Boss จะกด React 👍 (Approve) หรือ 👎 (Reject) บน Discord

### 🔴 Tier 3: OS Elevation (พื้นที่หวงห้าม - ต้องใช้ Password/Admin)
**ข้อห้ามขั้นเด็ดขาดสำหรับ AI (STRICTLY PROHIBITED TO AUTOMATE)**
- **ครอบคลุมถึง:** คำสั่งที่ขึ้นต้นด้วย `sudo`, การแก้ไขไฟล์ใน `/etc/`, `/usr/local/`, หรือ `C:\Windows`, การแก้ไข Registry หรือ Firewall
- **กฎของ AI:**
  - **ห้ามพยายามรันคำสั่งกลุ่มนี้ด้วยตัวเองเด็ดขาด** (เพราะจะทำให้ OS ระงับการทำงานและเด้งหน้าจอให้ผู้ใช้ใส่รหัสผ่านรัวๆ)
  - หากมีความจำเป็นต้องตั้งค่าระบบระดับ Global จริงๆ ให้ AI **"สร้างสคริปต์แยก"** ออกมา แล้วบอกให้ Developer นำสคริปต์นั้นไปรันด้วยตัวเองใน Terminal นอกการควบคุมของ AI

## 🛠️ Universal Environment Rules (การตั้งค่าโปรเจกต์)

เพื่อให้ AI ทำงานใน **Tier 0** ได้อย่างสมบูรณ์ โครงสร้างของโปรเจกต์ต้องปฏิบัติตามนี้:
1. **No Global Installs:** ห้ามติดตั้ง Dependencies ใดๆ ลงใน System Path โดยเด็ดขาด
2. **Containerization:** หากโปรเจกต์ต้องการ System Services (เช่น PostgreSQL, Redis, Message Brokers) **ห้าม** ติดตั้งลง OS โดยตรง ให้ใช้ `Docker` และ `docker-compose` เสมอ
3. **Local Ownership:** โฟลเดอร์โปรเจกต์ต้องมีสถานะเป็นของ User (Read/Write ได้โดยไม่ต้องใช้ sudo) หาก AI ตรวจพบปัญหา Permission Error ห้ามแก้ด้วย `sudo` แต่ให้แนะนำผู้ใช้รัน `chown` หรือ `icacls` เพื่อดึงกรรมสิทธิ์คืนกลับมาที่ User Space
