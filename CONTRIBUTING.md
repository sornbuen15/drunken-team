# Contributing to Drunken AGY Inn

First off, thank you for considering contributing to Drunken AGY Inn! It's people like you who make this toolkit better for everyone.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please report any unacceptable behavior to the project maintainers.

## How Can I Contribute?

### Reporting Bugs
* **Check existing issues/discussions** to see if the bug has already been reported.
* If it hasn't, open a new issue. Include a clear title, a description of the bug, steps to reproduce, and any relevant logs or screenshot links.

### Suggesting Enhancements
* Open a new discussion under the **Ideas** category in GitHub Discussions to brainstorm with the community.
* If the idea gains traction, we can formulate it into a structured task and track it.

### Submitting Pull Requests
We follow a strict development flow to keep the main code base stable:

1. **Fork the Repository:** Create a personal fork of the project.
2. **Branch from `develop`:** All new features or fixes MUST be branched from `develop` (not `main`).
   * Branch naming convention: `feature/<short-description>` or `bugfix/<short-description>`.
3. **Commit Messages:** Keep commit messages clean and descriptive.
4. **Submit a Pull Request (PR):** Target your PR to merge into the `develop` branch of this repository.
5. **Get Approval:** All PRs must receive at least 1 approving review from the repository owner (`sornbuen15`) before merging. Directly pushing to `develop` or `main` is protected.

Thank you for helping keep this project organized and secure!
